# LVGL 单片机移植指南：经典 Emoji 动效实现规范

本文档旨在指导 Agent 或嵌入式工程师，如何在资源受限的单片机（基于 LVGL 图形库）上，高保真地还原“经典 (Original)” Emoji 物理动效。

## 1. 设计原则与性能取舍

为了在单片机上实现 30~60 FPS 的流畅运行，本动效在设计上遵循以下原则：
* **禁止使用高算力滤镜**：彻底移除实时高斯模糊 (`filter: blur`) 和动态阴影 (`filter: drop-shadow`)。
* **基于矩阵变换的物理模拟**：所有生命力（软弹感、重量感）均通过纯粹的**缩放 (Scale)**、**旋转 (Rotation)** 和**透明度 (Opacity)** 的精确时间轴组合来实现。
* **层级结构 (Z-Order)**：从底到顶依次为：`眼眶背景 (黑)` -> `眼白 (白)` -> `瞳孔 (黑)` -> `Emoji 贴图`。

---

## 2. 全局环境与约束

### 2.1 动画周期
* **全局周期 (Cycle)**：3500ms (3.5s)
* **循环模式**：无限循环 (Infinite)

### 2.2 眨眼逻辑拦截（重要！）
* **规则**：在“经典”动效播放期间，**必须彻底屏蔽随机眨眼事件**。
* **原因**：眨眼动作会带来强制的全局 Y 轴物理挤压（Squash），这会打断/覆盖眼白与 Emoji 之间精心调校的同步缩放反馈。

## 4. LVGL 代码实现建议 (Timeline 模式)

建议使用 LVGL 的 `lv_anim_timeline_t`（动画时间轴）来管理这三个图层的复杂联动。
由于单片机运算资源宝贵，可以省略一些极小幅度的反弹帧（如 50% 时的 1.06 缩放），并利用 LVGL 内置的物理缓动曲线来自动补间：

1. **利用缓动函数**：对于 12% -> 20% -> 35% 这段最核心的“爆发 -> 砸底”过程，可尝试在 LVGL 中配置 `lv_anim_set_path_cb(&a, lv_anim_path_bounce)` 或 `lv_anim_path_overshoot`，通过硬件自动算出的物理回弹来替代密集打关键帧，这能极大降低代码量并提高流畅度。
2. **延迟函数**：眼白的 70ms 滞后，可通过 `lv_anim_set_delay(&a, 70)` 非常轻松地实现，无需重新计算关键帧时间戳。
3. **图像缓存**：Emoji 尽量使用编译进固件的 C 数组图像（如 `.c` 格式），避免在动画播放时进行实时的 PNG/SVG 解码。
