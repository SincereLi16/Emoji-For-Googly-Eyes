# N801 (Desk 419) 项目原则与长期记忆

## 核心协作原则
1. **修改前预读 (Pre-read before modification)**：在对任何文档（特别是 PRD 文档）进行修改、追加或重写之前，**必须**先使用读取工具（Read）查看文件的最新内容。
2. **保护用户修改 (Protect user edits)**：严格保存并继承用户在本地手动修改的所有内容，绝不能在未同步的情况下覆盖用户的内容。
3. **增量更新 (Incremental updates)**：在充分理解用户最新修改的基础上，进行针对性的增量更新。

## 业务与架构原则 (长期记忆)
1. **状态机路由逻辑 (优先级、打断与恢复)**：
   - 表情系统的切换不应基于“表情的静态视觉分类”，而必须基于**事件动作**和**抢占优先级**驱动。
   - **抢占优先级**：Level 3 (系统指令) > Level 2 (语音交互) > Level 1 (物理触摸) > Level 0 (发呆/音乐底态)。高优无条件抢占低优，低优被拦截或排队。
     - **Level 3 (系统指令)**：
       - **触发条件**：硬件按键中断（电源/蓝牙键）；网络配置周期（蓝牙/Wi-Fi）；全局异常保护触发。
       - **对应表情**：`400+ Command` 系列。
     - **Level 2 (语音交互)**：
       - **触发条件**：硅麦命中唤醒词或免唤醒指令，进入识别与回复周期。
       - **对应表情**：`500+ Hear` 系列。
     - **Level 1 (物理触摸)**：
       - **触发条件**：触控 IC 发生中断（短触 / 长摸）。
       - **对应表情**：`200+ Touch` 系列。
     - **Level 0 (底态)**：
        - **触发条件**：
          - `Daily`：电源`正常运行` 且 (蓝牙`未连接` 或 蓝牙`已连接 - 空闲`)。
          - `Music`：电源`正常运行` 且 蓝牙`已连接 - 播放`。
       - **对应表情**：`100+ Daily` 和 `300+ Music` 系列。
   - **切入与防错 (Kill-Timer)**：高优事件抢占时，必须**第一时间清除旧表情的倒计时定时器**，防止旧定时器到期引发状态错乱（诈尸）。
   - **切出与恢复 (记忆栈)**：高优表情退场时，必须平滑恢复到**被打断前的底态**（100+ 日常或 300+ 音乐）。退场条件分为“倒计时自动切出（固定时长的瞬态）”和“事件被动切出（无限循环，等待外部业务信号释放）”。
2. **底层渲染原则**：
   - 所有的表情切换（无论何种优先级）在底层图形渲染时，**必须**使用平滑过渡（Lerp 线性插值），严禁瞬间修改坐标导致跳帧。
