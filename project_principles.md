# N801 (Desk 419) 项目原则与长期记忆

## 核心协作原则
1. **修改前预读 (Pre-read before modification)**：在对任何文档（特别是 PRD 文档）进行修改、追加或重写之前，**必须**先使用读取工具（Read）查看文件的最新内容。
2. **保护用户修改 (Protect user edits)**：严格保存并继承用户在本地手动修改的所有内容，绝不能在未同步的情况下覆盖用户的内容。
3. **增量更新 (Incremental updates)**：在充分理解用户最新修改的基础上，进行针对性的增量更新。

## 业务与架构原则 (长期记忆)
1. **状态机路由逻辑 (优先级、生命周期与过渡)**：
   - **抢占优先级**：Level 4 (系统) > Level 3 (语音) > Level 2 (触摸) > Level 1 (底态)。高优无条件抢占，低优被拦截。
   - **动画生命周期**：
     - **底态 (Base State)**：无限循环（如 Daily 随机、Music 404）。无结束条件。
     - **瞬态 (One-Shot)**：单次定轨（如 歌词 300+、短触、单次提示）。倒计时归零结束。
     - **维持态 (Conditional)**：特殊瞬态（如 长按、聆听、网络搜索）。外部回调信号结束。
   - **过渡机制 (切入与切出)**：
     - **强行打断**：高优切入时，立刻清除定时器，100~200ms 强行交叉淡入。
     - **自然衔接**：底态内部轮播或瞬态/维持态结束时，等最后一帧播完再平滑流转。
2. **底层渲染原则**：
   - 所有的表情切换（无论何种优先级）在底层图形渲染时，**必须**使用平滑过渡（Lerp 线性插值），严禁瞬间修改坐标导致跳帧。
