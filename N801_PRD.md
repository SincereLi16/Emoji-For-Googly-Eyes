## 3. 硬件规格与定义 (Hardware Definition)

结合项目需求与量产验证基线，硬件选型与成本规划如下：

### 3.1 硬件元器件选型

#### 3.1.1 核心机构

- **主控芯片**：乐鑫 ESP32-S3-WROOM-1 (N16R8)，16MB Flash / 8MB PSRAM。
- **动作模组**：SG90 舵机（180度塑料齿轮），0\~36°限位，负载＞200g，体积＜20mm³。
- **整机尺寸**：87 × 89 × 78 mm。

#### 3.1.2 屏幕规格

- **屏幕型号**：GC9A01（双屏）。
- **核心参数**：1.28 英寸圆形 TFT 彩屏，物理尺寸 32.4×32.4mm，分辨率 240×240，30fps+，SPI 接口驱动。
- **亚克力透镜**：插接裸屏，外加定制圆形亚克力凸镜。

#### 3.1.3 声学配置

- **蓝牙接入**：杰理 AC6966B 芯片，支持 BLE 6.0 ＋ 经典蓝牙，有效传输距离 ≥10m。
- **音频解码**：ES7149 (DAC) ＋ 禾润 HTA8681 音频功放（D类 I2S），支持 A2DP 协议，SNR≥80dB。
- **喇叭单元**：4Ω 5W 喇叭（含被动盘），40×40mm 尺寸，输出功率 5W。
- **拾音输入**：MSM261D4030H1R 数字硅麦（1个），信噪比＞64dB，频响 20\~20Khz，表贴封装＜4mm²。

#### 3.1.4 交互硬件

- **触控输入**：单通道触控 IC，支持隔介质感应，触控面积≈15mm²。
- **物理按键**：2 个独立按键（1个电源键 ＋ 1个网络/蓝牙键）。
- **指示灯**：1个蓝牙指示灯 ＋ 1个 WIFI 指示灯 ＋ 1个电量指示灯。
- **外部接口**：1 个 Type-C 接口，支持 5V/2A 充电，**并支持 USB Audio Class（UAC）作为电脑有线声卡使用**。

#### 3.1.5 电源管理

- **电池规格**：103450型号 / 2000mAh 锂电池（带保护板及线材）。
- **充放电管理芯片**：IP5306-CK（常开不休眠版），支持 5V/2A 充电，支持 OVP 宽电压 28V 保护。

#### 3.1.6 结构与辅助物料

- **外壳结构件**：包含深黑色塑料机身、高反射喇叭装饰件、红色电源键、灰色蓝牙键及各类固定五金件。
- **PCBA**：主板 PCB 裸板、阻容贴片元件及相关内部连接排线。
- **包装与配件**：彩盒 ＋ 塑封包装，内附 80cm Type-C (5V/2A) 充电线（根据立项书标准）。

## 4. 软件功能与交互 (Software Functions & Interaction)

### 4.1 核心功能需求 (Core Functional Requirements)

#### 4.1.1 核心交互与信息流 (I/O)

为了确保软硬件边界清晰，N801 的核心功能基于“输入 (Input) - 处理 - 输出 (Output)”的工程维度进行定义：

1. **输入信息 (Input)**
   - **触觉输入**：通过顶部 **单通道触控 IC** 采集用户的短触与长摸信号，用于触发情绪互动或控制音乐播放。
   - **听觉输入**：通过 **数字硅麦 (MSM261)** 实时监听环境音与特定唤醒词（如“Hey Googly”），用于语音控制。
   - **音频流输入**：通过 **蓝牙天线/Type-C 接口** 接收外部设备推流的高保真音频数据。
   - **网络层输入**：通过 **Wi-Fi 模块 (ESP32)** 接收云端下发的音乐 Metadata 与专属表情包资源。
   - **物理控制**：通过 2 个物理按键（红键/灰键）采集用户的电源控制与蓝牙 / 网络配置指令。
2. **输出反馈 (Output)**
   - **视觉输出**：通过机身正面的 **两块圆形 TFT 屏幕 (GC9A01)** 渲染 30fps+ 的动态表情。
   - **听觉输出**：通过内部的 **DAC (ES7149) 与 4Ω 5W 喇叭** 播放高品质音乐及系统互动音效。
   - **动觉输出**：通过内部的 **舵机 (SG90)** 驱动机身在不同电源状态下的物理起伏。
   - **状态输出**：通过 3 颗隐藏式指示灯（蓝/白/红绿）直观反馈底层的连接与电量状态。

#### 4.1.2 底层设备状态模型 (Device State Model)

设备在软件架构层面维护三个独立的底层服务状态。当这些状态发生变化时，将触发上层应用模式的切换。

1. **电源状态 (Power)**
   - **深度关断**：整机断电，所有模块硬关断。仅可通过长按电源键唤醒。
   - **待机休眠**：低功耗休眠。屏幕与功放关闭，仅保留麦克风与触摸板 IC 监听。
   - **正常运行**：系统全功能开启。舵机、主控、双圆屏、音频功放、麦克风及触控模块均处于工作或就绪状态。
2. **蓝牙状态 (Bluetooth)**
   - **未连接**：蓝牙模块空闲，无数据交互。
   - **待配对 / 搜索**：蓝牙天线开启广播，等待外部设备发起连接。
   - **已连接**：蓝牙音频链路（A2DP）握手成功，系统就绪等待推流。
   - **音乐播放中**：持续接收并解码外部高保真音频流，驱动音频功放工作。
3. **网络状态 (Wi-Fi)**
   - **离线**：Wi-Fi 模块休眠或未配置，无网络连接。
   - **配网中**：开启配网模式广播，等待 App 写入网络配置凭证（SSID / 密码）。
   - **在线**：成功配网并接入云端API，开启后台静默校验，请求歌曲 Metadata 及 Emoji 文件。

#### 4.1.3 硬件交互 (Hardware Buttons)

极简设计下保留 3 个输入区，功能与按压时长严格绑定：

- **电源键**
  - **短按 (≤1s)**：在 **待机休眠** 与 **正常运行** 间切换。
  - **长按 (3s)**：在 深度关机 与 **正常运行** 间切换。
- **蓝牙 / 网络键**
  - **短按 (≤1s)**：未连接 - **搜索配对**蓝牙 / 已连接 - **断开**蓝牙。
  - **长按 (3s)**：未配网 - 进入 **Wi-Fi 配网** / 已配网 - **断开并清除** Wi-Fi 记录。
- **触摸键**&#x20;
  - **待机休眠**：**长摸 (3s)**，**唤醒**设备进入正常运行态。
  - **运行中 - Daily 模式**：短触 / 长摸，触发机身**互动表情**。
  - **运行中 - Music 模式**：短触，**播放 / 暂停**当前音乐。

#### 4.1.4 执行器与传感器状态 (Actuators & Sensors)

- **舵机**&#x20;
  - **正常运行**：通电锁止，保持机身处于 **抬头固定 36°** 姿态。
  - **休眠 / 深度关机**：断电卸力，机身回收-36°至 **低头 0°** 姿态。
- **麦克风**&#x20;
  - **待机休眠 / 正常运行**：标准唤醒模式，接收唤醒词＋指令。麦克风保持低功耗监听，必须通过唤醒词 Hey Googly 触发，随后接收设备控制与查询类常规指令。
  - **音乐模式**：特权唤醒模式，仅接收简短指令。系统主动挂起 Hey Googly 常规唤醒词。仅在本地加载极简的**免唤醒快捷指令**，用户无需唤醒即可直接打断。
- **双圆屏**
  - **正常运行**：全量加载 30fps+ 的动态表情 UI。
  - **待机休眠 / 关机**：切断背光并熄屏。

#### 4.1.4 指示灯反馈系统

- **蓝牙指示灯 - 蓝光**：慢闪（未连接） / 常亮（已连接）。
- **WIFI 指示灯 - 白光**：熄灭（未连接）/ 闪烁（连接中） / 常亮（已连接）。&#x20;
- **电量指示灯 - 红 / 绿光**：熄灭（正常电量）/ 红灯闪烁（低电量） / 红灯常亮（充电中） / 绿灯常亮（已充满）。

> **补充说明：充电交互**
> 插入 Type-C 充电时为**静默充电**，仅通过侧边电量指示灯进行状态提示。屏幕**不显示**任何特定的充电动画。

#### 4.1.5 全局异常与保护机制 (Global Protection)

为确保硬件安全与用户体验的连贯性，系统在底层配置了跨状态的强制保护机制：

1. **极低电保护 (Battery Protection)**：当检测到电池电压 < 3.3V 时，抛出最高优先级中断。播报低电提示音后，强制切断所有功耗模块并进入深度关机状态，防止电池过放。
2. **舵机堵转保护与恢复 (Stall Protection & Recovery)**：在机身抬头 / 起伏过程中，若检测到舵机电流异常增大（如被用户强行按压阻挡），系统立即卸力放弃当前动作，进入低头保护状态，防止烧毁电机或拉低系统电压。

   触发舵机堵转保护后，系统将等待用户的**下一次主动输入**作为重置信号来触发主动复位抬头。有效的触发输入包括：
   - **触觉输入**：用户短触或长摸触摸区。
   - **听觉输入**：成功识别到唤醒词（如“Hey Googly”）。
   - **状态切换**：蓝牙连接成功并开始播放音乐，触发进入 MUSIC 模式。
   - **物理按键**：通过短按电源键从休眠唤醒，或长按电源键重新开机，进入正常运行状态。
3. **断网与坏包兜底 (Offline Fallback)**：当 Wi-Fi 突然断开，或云端表情包下载失败 / 文件校验不完整时，系统静默回退。仅在当前歌曲播放期间，仅调用本地 Flash 中的 “播放”动画循环，直到切换至下一首歌时，重新尝试正常的云端链路请求与校验。
4. **高温与防破音保护 (Audio & Thermal Protection)**：在大音量连续播放导致功放温度异常，或播放极端音频可能撕裂振膜时，DSP 与功放启动动态限幅（AGC）并主动降低增益，牺牲部分听感以保护声学硬件。

### 4.2 语音指令交互设计 (Voice Command Interaction)

#### 4.2.1 技术实现基准 (Technical Baseline)

N801 不具备开放域对话（Chatbot）功能，语音交互仅作为**设备控制**与**获取轻量化生活信息**的快捷入口。
为了实现毫秒级的极速响应并节省云端 API 成本，系统采用基于 ESP-SR 框架的端侧离线关键词匹配（MultiNet）方案。

- **实现原理**：将各类用户意图穷举为多个高频命令词，直接烧录在本地芯片中。当用户发音命中关键词表中任意一词时，系统在本地直接映射为对应意图并执行指令。

#### 4.2.2 标准唤醒指令库

在 DAILY (日常模式) 下，用户需先使用唤醒词“Hey Googly”唤醒设备，随后下发预设指令。以下为核心意图分类及反馈逻辑。

| 意图分类      | 意图 (Action) | 命令词示例  | 设备反馈与动作        |
| :-------- | :---------- | :----- | :------------- |
| **设备与网络** | 蓝牙配对        | “打开蓝牙” | 蓝牙广播 / UI / 灯效 |
| <br />    | 关闭蓝牙        | “断开蓝牙” | 蓝牙断连 / UI / 灯效 |
| <br />    | 设备休眠        | “进入休眠” | 舵机闭合 / UI / 灯效 |
| <br />    | 电量查询        | “有多少电” | 状态查询 / UI      |
| **生活与工具** | 时间查询        | “现在几点” | 时钟查询 / UI      |

### 4.3 情绪与交互系统 (Emotion & Interaction System)

#### 4.3.1 屏幕动画与模式仲裁 (Mode Manager)

作为“桌面上的生命”，双圆屏是 N801 情绪展示的核心窗口。系统根据触发时机，将设备状态与屏幕动画划分为**三大应用模式**，并严格仲裁底层硬件资源的使用权。

**1. 模式定义与视觉表现**

- **DAILY 模式**：正常运行且**无音乐播放**时（未连接 / 已连接未播放）。屏幕主要展示闲置陪伴表情（如眨眼、东张西望、打瞌睡）及触摸反馈表情。
- **MUSIC (音乐模式)**：正常运行且**音乐播放中**。屏幕主要展示音乐专属表情（如音符眼）与云端拉取的专属歌曲 Emoji。注：DAILY 与 MUSIC 互为平行的主模式，完全互斥。
- **COMMAND (指令模式)**：**语音互动或系统提示期间**。作为高优先级临时覆盖层（Overlay），一旦唤醒或发生系统事件，将瞬间接管屏幕展示“聆听/思考”或“系统图标”，结束后**立刻释放屏幕权，恢复至原状态**。

**2. 传感器与资源状态仲裁**

| 应用模式        |  蓝牙音乐链路  |  麦克风 (听觉)  |   触摸区感应 (触觉)  |
| :---------- | :------: | :--------: | :-----------: |
| **DAILY**   |  关闭 / 待机 |  **保持监听**  |   **触发互动表情**  |
| **MUSIC**   | **独占播放** | **仅听播放控制** | **控制播放 / 暂停** |
